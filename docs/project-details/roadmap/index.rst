نقشه راه پروژه
===============

برای مشاهده برنامه‌های بلندمدت پروژه، به نقشه راه پروژه در مخزن کد منبع مراجعه کنید:

https://github.com/educates/educates-training-platform/blob/develop/developer-docs/project-roadmap.md

جزئیات برنامه‌های کوتاه‌مدت‌تر در ادامه فهرست شده‌اند.

.. _upcoming-changes:

تغییرات پیش‌رو
---------------

توجه داشته باشید که قابلیت‌های زیر منسوخ (Deprecated) شده‌اند و طبق برنامه فعلی، در سری نسخه‌های 3.X پلتفرم Educates حذف خواهند شد:

* رندرر کلاسیک برای دستورالعمل‌های کارگاه (Markdown و AsciiDoc) حذف خواهد شد.  
  تمامی کارگاه‌ها باید به رندرر مبتنی بر Hugo (Markdown) منتقل شوند.

* مکانیزم قدیمی دانلود فایل‌های کارگاه از طریق مشخص کردن  
  ``spec.content.files`` در تعریف کارگاه حذف خواهد شد.  
  به جای آن باید از مکانیزم مبتنی بر ``vendir`` استفاده شود.

* کنسول وب Octant برای مشاهده و تعامل با کلاستر Kubernetes حذف خواهد شد.  
  به جای آن باید از داشبورد استاندارد Kubernetes استفاده شود.

* استفاده از فایل‌های ``profile.d`` با افزودن متغیرهای محیطی به فایل  
  ``WORKSHOP_ENV`` از طریق اسکریپت‌های ``setup.d`` جایگزین شده است.  
  پشتیبانی از ``profile.d`` حذف خواهد شد تا امکان پیاده‌سازی مجدد اسکریپت‌های
  مقداردهی اولیه کانتینر کارگاه به‌صورت یک برنامه مستقل Go فراهم شود.  
  کارگاه‌ها باید تنظیم متغیرهای محیطی را از طریق اسکریپت‌های ``setup.d`` انجام دهند.

* پشتیبانی از استقرار Educates روی کلاسترهایی که از Pod Security Policies
  استفاده می‌کنند حذف خواهد شد.

* ارائه بسته‌های Carvel Repository برای Educates متوقف خواهد شد.  
  همچنان امکان نصب Educates از طریق بسته‌های Carvel با استفاده از منابع ``App``
  وجود خواهد داشت، اما نوع منبع ``PackageRepository`` که دسترسی به چند نسخه را
  فراهم می‌کرد حذف خواهد شد.  
  این تصمیم به این دلیل گرفته شده که به‌روزرسانی درجا (Rolling Update) نسخه
  Educates همیشه امکان‌پذیر نیست و توصیه می‌شود به جای ارتقاء، نصب مجدد انجام شود.  
  **این تغییرات در Educates 3.0.0 اعمال شده‌اند.**

* ابزار خط فرمان ``skaffold`` حذف خواهد شد.  
  در صورت نیاز به این ابزار در یک کارگاه خاص، باید در فرآیند راه‌اندازی کارگاه نصب شود.

* ابزار خط فرمان ``buildah`` مدتی است که ارائه شده اما به دلیل محدودیت‌های
  اکثر محیط‌های Kubernetes احتمالاً عملکرد مناسبی ندارد.  
  در صورت تأیید این موضوع، حذف خواهد شد.

* نام کاربر UNIX مربوط به Session کارگاه از ``eduk8s`` به نامی غیرمرتبط با
  نام پروژه تغییر خواهد کرد.  
  کارگاه‌ها باید تا حد امکان از مسیرهای ``~/`` یا ``$HOME/`` استفاده کنند،
  اما در مواردی که امکان استفاده از این میانبرها وجود ندارد، باید مسیرها
  به‌روزرسانی شوند.

سایر تغییرات مهم در نسخه 3.X
-----------------------------

سایر تغییرات قابل توجه که برای نسخه 3.X برنامه‌ریزی شده‌اند عبارت‌اند از:

* قابلیت ابزار CLI با نام ``educates`` برای نصب Educates و ایجاد کلاستر محلی
  Kind جهت میزبانی آن، بازطراحی شده است.  
  تجربه کاربری مشابه باقی خواهد ماند، اما نیاز به وجود ``kapp-controller``
  در کلاستر Kubernetes حذف می‌شود.  
  بسته ``kapp-controller`` همچنان می‌تواند به‌صورت اختیاری نصب شود،
  زیرا ممکن است برخی کارگاه‌ها به آن وابسته باشند.  
  CLI همچنین از نصب‌های پیشنهادی (Opinionated Install) روی ارائه‌دهندگان IaaS
  مانند AWS، GCP و Azure پشتیبانی خواهد کرد.  
  **این تغییرات در Educates 3.0.0 اعمال شده‌اند.**

* پشتیبانی سطح اول (First-class support) از OpenShift که قبلاً حذف شده بود،
  دوباره به Educates افزوده خواهد شد.  
  این شامل پشتیبانی از استفاده از کنسول وب OpenShift به‌صورت تب داشبورد،
  و همچنین اعمال RBAC برای منابع Namespaced خاص OpenShift خواهد بود.
